## Slide 4

- Problem: CUDA out of memory

- The orthomosaic image size is very large ( 10742 x 11102 pixels).

- Models like DeepLabV3 struggle to process such high-resolution images in a single pass due to GPU memory constraints.

- Why Does This Happen?

- When the entire image is loaded, it exceeds GPU memory limits.

- PyTorch Tensor Size: A (1, 3, 11102, 10742) tensor requires multiple GBs of VRAM, causing CUDA Out of Memory (OOM) errors.

- GPU has to handle both model weights and image data, leading to insufficient free memory.

## Slide 5

- Solution: Tiling

- Divides the image into smaller tiles (e.g., 1024x1024 or 2048x2048 pixels).

- Each tile is processed independently, keeping resolution intact.

- Parallel processing speeds up execution.

- No loss of information, unlike resizing.

- Split the orthomosaic into smaller tiles.

- Process each tile individually for segmentation

- Merge the results to reconstruct the full segmented image.

## Slide 6

- Potential problem with tiling?

- Deepforest currently works on our full orthomosaic, but this might not be the case as the area increases

- Objects at tile boundaries may get cut off, leading to incomplete detections.

- Trees near the edges of a tile may not be fully detected.

- May cause duplicate detections if a tree is split between two tiles.

- We can keep segmentation tiled and crown detection on full orthomosaic after applying stitched segmentation mask; force CPU usage if CUDA out of Memory

- We can use overlapping tiles (e.g., 10-20% overlap) to preserve context and have a post-processing step to merge overlapping results

## Slide 11

- Detectree2

- Open-source Python library designed for tree segmentation and detection in aerial or drone imagery specifically

- Tailored to handle drone-captured orthomosaics, providing high accuracy in detecting individual trees and canopy boundaries.

- Offers multiple methods like semantic segmentation (e.g., DeepLab), NDVI-based segmentation, clustering, or combined approaches

- Also has some Geospatial Integration

- Pre-trained models are already tuned for vegetation and tree detection

## Slide 12

- Standard Workflow with detectree2

- Tile the orthomosaics and crown data (for training, validation and testing)

- Train (and tune) a model on the training tiles

- Evaluate the model performance by predicting on the test tiles and comparing to manual crowns for the tiles

- Using the trained model to predict the crowns over the entire region of interest

- They have provided some example jupyter notebooks,

- https://github.com/PatBall1/detectree2/blob/master/notebooks/colab/

## Slide 14

- Datastructure, now ready using deepforest

DeepForest-based Tree Detection
• Uses drone-derived orthomosaics with a pre-trained DeepForest model
• Detects individual trees with bounding boxes, confidence scores, and labels

Data Structures & Tracking
• Detection: Stores metadata (run_id, bbox, center, score, label) for each detected tree
• Tree: Aggregates multiple detections of the same tree across runs
• TreeTracker: Uses Euclidean distance matching (with a configurable threshold) to track trees over time

Orthomosaic Processing
• Reads georeferenced images with rasterio
• Annotates detections on images for visual verification
• Modular design enables easy expansion for additional processing

State Persistence
• Saves the tracker state using pickle
• Allows cumulative tracking across multiple orthomosaics without reprocessing previous runs

- Once the segmentation pipeline is ready, the detection class can be easily modified to integrate
