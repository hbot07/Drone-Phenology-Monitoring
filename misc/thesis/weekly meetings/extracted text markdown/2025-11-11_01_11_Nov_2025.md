## Slide 2

- Behind Detectree2: How It Works

- Detectree2 is an open-source deep learning model for detecting and delineating individual tree crowns from high-resolution aerial RGB images.

- It is built on Mask R-CNN (via Detectron2) — a Convolutional Neural Network (CNN) architecture for instance segmentation.

- The model pipeline:

- Feature Extraction: CNN (ResNet + FPN backbone) learns visual features (edges, colours, textures, shapes).

- Region Proposal Network (RPN): Identifies potential tree crown regions.

- Mask Head: Produces a pixel-level crown segmentation mask for each detected tree.

- Output: We get detected crowns

## Slide 3

- Features Used by Detectree2 to Detect Crowns

- 1.)Spectral (Colour)

- Differences in foliage colour, greenness, and brightness; contrast between crown and background.

- 2.)Texture

- Leaf and branch density, internal variation, roughness of crown surface.

- 3.)Shape / Edge

- Irregular crown outlines, curvature, and clear boundary edges.

- 4.)Scale / Size

- Typical crown dimensions and scale patterns from the feature pyramid network.

- 5.)Spatial Context

- Separation between neighbouring crowns, canopy gaps, and adjacency patterns.

- 6.)Shadow / Lighting

- Brightness gradients and internal or cast shadows that outline crowns.

- Crowns have overlap within OM

- Detectree2 merges crowns above the iou_threshold we set(they’re considered the same tree; ), these are crowns which did not satisfy that threshold.

- Pruning rule:

- We select the larger crown if the candidates have a max(overlap_cur, overlap_next) > 0.7
