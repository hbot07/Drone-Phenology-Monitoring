## Slide 2

- Conditional Threshold Matching

- Every crown pair within a base radius (base_max_dist) gets per-pair features from _compute_pair_metrics: IoU, overlap ratios, centroid distance, area ratio, geometry containment flags, and a baseline similarity

- Low-signal pairs are dropped early via min_base_similarity and overlap_gate

## Slide 3

- containment: one geometry fully contains the other.

- one_to_one: both crowns overlap strongly and are unique partners.

- split / merge: a previous crown overlaps multiple current crowns (split) or vice versa.

- partial_overlap: moderate IoU or overlap without strong mutual exclusivity.

- proximity: fallback when overlap is weak but centroids fall within a size-scaled radius.

- Anything failing all gates is tagged none and removed.

- Classification

## Slide 4

- Case Specific Processing

- Cases are processed in priority order (one_to_one → containment → split → merge → partial_overlap → proximity).

- We have defined a function for match deciding for each of these specific cases

- Or we can define specific thresholds and weights for each case and toggle repeating logics on or off(next slide)
