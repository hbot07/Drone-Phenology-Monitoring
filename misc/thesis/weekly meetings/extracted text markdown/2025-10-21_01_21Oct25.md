## Slide 3

- Shuffle Connectivity

- For consecutive orthomosaics, we gather votes from each shuffle

- If these 2 nodes are connected in a shuffle, vote += 1

- We select the edge which has the maximum votes

- Across all the shuffles, we also maintain a list for every node

- This list contains all the nodes this nodes is connected to

- If there is no edge leaving the current node, we check if this list has any node from the next orthomosaic in the get_tracking() function

## Slide 4

- Rotational Shuffles

- We run the conditional matching algorithm on all the rotational shuffles of the orthomosaics

- Eg: 1-2-3-4, 4-1-2-3, 3-4-1-2, 2-3-4-1

- get_chains(): get a list of the chains that tree is a part of in all of the shuffles

- We select the longest chain out of these, and sort according to time at which the orthomosaic was taken to get the matching chain

## Slide 7

- Virtual Edges

- These are edges between any two pairs of orthomosaics, they do not need to be consecutive

- We iterate over the orthomosaics, and compare each new orthomosaic which all of the previous ones we have

- We use the same conditional matching approach as earlier for creating virtual edges as well

- If there is no edge leaving a node, we see if the node has a outgoing virtual edge connecting it to a node.

- In get_tracking() we see the ancestors of this node(with both virtual and real edges) and select the chain greedily based on similarity scores

## Slide 8

- Problem

- We are getting wrong matches as the errors propagate

- We see that this algorithm produces chains with edges between nodes that have similarity scores lower than our set threshold
