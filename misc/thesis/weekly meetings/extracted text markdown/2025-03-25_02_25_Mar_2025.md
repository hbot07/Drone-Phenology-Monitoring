## Slide 7

- PCC

- Global PCC looks at the whole overlap and tries to find one dominant translation

- estimating one shift from the whole overlap is causing problem(global PCC can lock onto the wrong structure and return a bad shift)

- There are partial overlap differences and repeated textures

- Idea: Estimate many shifts from many local tiles and then aggregate them. Even if some tiles are confusing or changed a lot, many tiles still carry correct alignment signal.

## Slide 8

- PCC-Tiled

- Split the overlap into a grid of tiles

- For each tile, run PCC independently to get a per-tile shift

- Aggregate the non-rejected tile shifts by taking median

- Tile rejection:

- High error: PCC returns an “error”/quality metric. If too high, reject. This is to reject tiles where PCC has no confident peak.

- Too-large-shift: reject tiles voting for very large shifts relative to the overlap size.
