## Slide 4

- problem recognised

- In the image, several red crowns overlap with the blue crown.

- Despite the visual overlap, the calculated IoU remains low (often < 0.4 or even < 0.3) because the intersection area is small compared to the union area.

- Due to this issue, certain crowns (e.g., C6 in above graph) cannot be detected or visualized properly.

- To address this, we propose using multiple IoU thresholds:

- If there is an overlap and its IoU is above 0.7, we select that crown and ignore the rest.

- If there are multiple overlaps within a threshold range (e.g., 0.4–0.7), then we treat that crown as split.

- Or if you have any other idea to resolve this we can also follow that.
